macOS 10.15 Catalina 于 2019.6.3 在苹果全球开发者大会（WWDC）发布 而我在 6.4 才更新
目前可通过安装描述文件以体验最新系统

（碎碎念：最近可能会写一篇小应用推荐的说 咕咕咕

更新

支持机型

macOS Catalina 与 macOS Mojave 所支持的机型相同，也就是说，只要能升级 Mojave 的都可以体验&升级最新系统

2015 款 MacBook 及之后机型
2012 款 MacBook Air 及之后机型
2012 款 MacBook Pro 及之后机型
2012 款 Mac mini 及之后机型
2012 款 iMac 及之后机型
2013 款 Mac Pro 及之后机型
所有的 iMac Pro 机型

如何更新 ？

2020.01.20 更新

在系统偏好设置中的软件更新，点击检查更新即可

macOS 10.15 Catalina 需要安装“描述文件”以更新
点击以下载描述文件

注意：此版本为第一个 Beta 版，可能存在某些不稳定（虽然我体验良好）

将所下载的文件解压，双击运行，同意协议，根据提示输入密码后即可安装描述文件（废话）

安装完描述文件后，打开设置，点击软件更新便可以收到更新
下载安装即可

升级之前强烈建议使用时间机器（Time Machine）先备份后升级
一定不要学我这种头铁的直接升级

官方介绍：

https://www.apple.com/macos/catalina/

一些说明

此版本 macOS 10.15 Catalina 只支持 64 位应用程序，如有需要使用到 32 位应用程序，请勿升级！

主要变化

系统

Catalyst

一个全新的开发平台（与我无关

Gatekeeper

Dedicated system volume
macOS Catalina runs in its own read-only volume, so it’s separate from all other data on your Mac, and nothing can accidentally overwrite your system files. And Gatekeeper ensures that new apps you install have been checked for known security issues before you run them, so you’re always using good software.

大概意思就是系统和用户数据分开存储，系统分区是只读的（system 分区只读，好熟悉的赶脚)
估计一些软件要凉凉了

图中的 Mac OS 被分为了“系统”“数据”

其它变化

终端

首先来看一条资讯

macOS Catalina 默认用 zsh 替代 Bash

苹果即将释出的新一代操作系统 macOS Catalina 将默认用 zsh 替代 Bash 。Bash 仍然可用。但苹果已经示意 macOS Mojave 或更早版本的开发者开始准备迁移到 zsh，因为 Bash 最终将会移除。苹果没有解释它为什么要怎么做，推测与 Bash 使用的许可证有关。macOS 使用的 Bash 版本长期停留在采用 GPLv2 的 Bash 3.2，而不是较新的采用 GPLv3 的版本。苹果一直避免使用 GPLv3 软件包，因为它对苹果这样的公司有更多的限制，它的条款包含了明确的专利授权。Media

https://www.solidot.org/story?sid=60882

使用体验

提示：这是以我的笔记本“2012 款 MacBook Air”为基础的新系统体验
系统版本：10.15 Beta 1 19A471t

卡顿情况：该卡的还是会卡（老电脑性能抓急哇），但是 chrome 会时不时的抽下风
使用情况：感觉良好，速度有所提升，可以升级，但有些小问题，瑕不掩瑜（我自己使用的软件全部正常使用

小问题 1：刚刚升级后，发现耳机无法连接使用
解决办法：开机时按住 option+Command+P+R 在两次启动声之后松开这些按键
官方文档

小问题 2：Dock 栏 图标以及 启动台 中的应用图标无法刷新（在更新应用图标后）
解决办法：终端输入
